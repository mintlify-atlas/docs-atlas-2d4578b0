This guide demonstrates how to implement full CRUD (Create, Read, Update, Delete) functionality using Symfony's controller system, forms, and Doctrine ORM.

## Task Controller

The `TaskController` handles all CRUD operations for the Task entity:

```php src/Controller/TaskController.php
<?php

namespace App\Controller;

use App\Entity\Task;
use App\Form\TaskType;
use App\Repository\TaskRepository;
use Doctrine\ORM\EntityManagerInterface;
use Symfony\Bundle\FrameworkBundle\Controller\AbstractController;
use Symfony\Component\HttpFoundation\Request;
use Symfony\Component\HttpFoundation\Response;
use Symfony\Component\Routing\Attribute\Route;

#[Route('/task')]
final class TaskController extends AbstractController
{
    #[Route(name: 'app_task_index', methods: ['GET'])]
    public function index(TaskRepository $taskRepository): Response
    {
        return $this->render('task/index.html.twig', [
            'tasks' => $taskRepository->findAll(),
        ]);
    }

    #[Route('/new', name: 'app_task_new', methods: ['GET', 'POST'])]
    public function new(Request $request, EntityManagerInterface $entityManager): Response
    {
        $task = new Task();
        $form = $this->createForm(TaskType::class, $task);
        $form->handleRequest($request);

        if ($form->isSubmitted() && $form->isValid()) {
            $entityManager->persist($task);
            $entityManager->flush();

            return $this->redirectToRoute('app_task_index', [], Response::HTTP_SEE_OTHER);
        }

        return $this->render('task/new.html.twig', [
            'task' => $task,
            'form' => $form,
        ]);
    }

    #[Route('/{id}', name: 'app_task_show', methods: ['GET'])]
    public function show(Task $task): Response
    {
        return $this->render('task/show.html.twig', [
            'task' => $task,
        ]);
    }

    #[Route('/{id}/edit', name: 'app_task_edit', methods: ['GET', 'POST'])]
    public function edit(Request $request, Task $task, EntityManagerInterface $entityManager): Response
    {
        $form = $this->createForm(TaskType::class, $task);
        $form->handleRequest($request);

        if ($form->isSubmitted() && $form->isValid()) {
            $entityManager->flush();

            return $this->redirectToRoute('app_task_index', [], Response::HTTP_SEE_OTHER);
        }

        return $this->render('task/edit.html.twig', [
            'task' => $task,
            'form' => $form,
        ]);
    }

    #[Route('/{id}', name: 'app_task_delete', methods: ['POST'])]
    public function delete(Request $request, Task $task, EntityManagerInterface $entityManager): Response
    {
        if ($this->isCsrfTokenValid('delete'.$task->getId(), $request->getPayload()->getString('_token'))) {
            $entityManager->remove($task);
            $entityManager->flush();
        }

        return $this->redirectToRoute('app_task_index', [], Response::HTTP_SEE_OTHER);
    }
}
```


    **Workflow:**
    <Steps>

## Form Handling

The `TaskType` form class defines the form structure:

```php src/Form/TaskType.php
<?php

namespace App\Form;

use App\Entity\Task;
use Symfony\Component\Form\AbstractType;
use Symfony\Component\Form\FormBuilderInterface;
use Symfony\Component\OptionsResolver\OptionsResolver;

class TaskType extends AbstractType
{
    public function buildForm(FormBuilderInterface $builder, array $options): void
    {
        $builder
            ->add('title')
            ->add('description')
        ;
    }

    public function configureOptions(OptionsResolver $resolver): void
    {
        $resolver->setDefaults([
            'data_class' => Task::class,
        ]);
    }
}
```

### Form Features

<Steps>
  <Step title="Form Builder">
    `buildForm()` defines which entity properties to include as form fields
  </Step>
  
  <Step title="Data Binding">
    `data_class` binds the form to the Task entity, enabling automatic data mapping
  </Step>
  
  <Step title="Field Type Detection">
    Symfony automatically detects field types from entity property types:
    - `string` properties become text inputs
    - `TEXT` columns become textarea fields
  </Step>
  
  <Step title="Validation">
    Form validation uses entity constraints (can be added via attributes on the entity)
  </Step>
</Steps>

<Note>
The form builder uses method chaining for clean, readable code. Each `add()` call returns the builder instance.
</Note>

### Built-in Repository Methods

All Doctrine repositories inherit these methods:

| Method | Description | Example |
|--------|-------------|--------|
| `find($id)` | Find by primary key | `$repo->find(1)` |
| `findAll()` | Find all entities | `$repo->findAll()` |
| `findBy()` | Find by criteria | `$repo->findBy(['status' => 'active'])` |
| `findOneBy()` | Find one by criteria | `$repo->findOneBy(['title' => 'Test'])` |
| `count()` | Count entities | `$repo->count(['status' => 'done'])` |

### Custom Query Methods

You can add custom query methods to your repository:

```php
public function findActiveTasks(): array
{
    return $this->createQueryBuilder('t')
        ->andWhere('t.status = :status')
        ->setParameter('status', 'active')
        ->orderBy('t.createdAt', 'DESC')
        ->getQuery()
        ->getResult();
}
```

**Query Builder Benefits:**
- Type-safe query construction
- IDE auto-completion support
- Protection against SQL injection
- Reusable query components

## Dependency Injection

Symfony's dependency injection provides required services:

```php
// Inject repository
public function index(TaskRepository $taskRepository): Response

// Inject entity manager
public function new(Request $request, EntityManagerInterface $entityManager): Response

// Inject both entity and request
public function edit(Request $request, Task $task, EntityManagerInterface $entityManager): Response
```

**Auto-wiring Benefits:**
- No manual service configuration needed
- Type-hinted parameters automatically resolved
- Easy to test with dependency injection
- Clean, readable controller code

## Entity Manager Operations

The `EntityManagerInterface` provides core database operations:

<CodeGroup>

```php Persist (Insert)
// Mark new entity for insertion
$entityManager->persist($task);
$entityManager->flush(); // Execute INSERT
```

```php Update
// No persist needed for managed entities
$task->setTitle('Updated Title');
$entityManager->flush(); // Execute UPDATE
```

```php Remove (Delete)
// Mark entity for deletion
$entityManager->remove($task);
$entityManager->flush(); // Execute DELETE
```

```php Transaction
// Multiple operations in one transaction
$entityManager->persist($task1);
$entityManager->persist($task2);
$entityManager->flush(); // Single transaction
```

</CodeGroup>

<Warning>
Always call `flush()` to execute database operations. Changes are only persisted when flush() is called.
</Warning>

## Best Practices

<Steps>
  <Step title="Use HTTP Methods Correctly">
    - GET for reading data
    - POST for creating and deleting (with CSRF)
    - Use semantic HTTP status codes (303 for redirects)
  </Step>
  
  <Step title="CSRF Protection">
    Always validate CSRF tokens for state-changing operations (create, update, delete)
  </Step>
  
  <Step title="Repository Pattern">
    Keep database queries in repositories, not controllers. This separates concerns and improves testability.
  </Step>
  
  <Step title="Form Handling">
    Use Symfony forms for validation and CSRF protection rather than manual request handling
  </Step>
  
  <Step title="Entity Manager">
    Call `flush()` only once per request when possible to minimize database transactions
  </Step>
</Steps>
