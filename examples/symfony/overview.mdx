## Basic Controller Example

The repository includes a simple controller example (`LuckyController`) that demonstrates basic Symfony routing and responses:

```php LuckyController.php
<?php
namespace App\Controller;

use Symfony\Component\HttpFoundation\Response;
use Symfony\Component\Routing\Attribute\Route;

class LuckyController
{
    #[Route('/lucky/number')]
    public function number(): Response
    {
        $number = random_int(0, 100);

        return new Response(
            '<html><body>Lucky number: '.$number.'</body></html>'
        );
    }
}
```

This example demonstrates:
- **PHP Attributes for Routing** - Using `#[Route]` attribute to define routes
- **Simple Response** - Returning HTML content directly without a template
- **Clean Controller Syntax** - Minimal code for a working endpoint

<Info>
  This is a basic example showing Symfony routing fundamentals. For production applications, use Twig templates as demonstrated in the Task Manager example.
</Info>

## Technology Stack

The examples use the following modern technologies:

- **Symfony 8.0** - Latest version of the Symfony framework
- **PHP 8.4** - Modern PHP with attributes and type safety
- **Doctrine ORM 3.6** - Object-relational mapping for database operations
- **FrankenPHP** - Modern PHP application server built on Caddy
- **PostgreSQL 15** - Relational database
- **Docker & Docker Compose** - Container orchestration
- **Twig** - Template engine for views
