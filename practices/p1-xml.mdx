## Objective

In this practice assignment, you will create a well-formed XML document that structures information in a hierarchical format. This exercise will help you understand XML syntax, document structure, and the importance of proper formatting in markup languages.

<Info>
  XML (eXtensible Markup Language) is a markup language that defines rules for encoding documents in a format that is both human-readable and machine-readable.
</Info>

## Example XML Structure

Here's an example of a well-formed XML document to guide you:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<curso>
  <informacion>
    <nombre>Programación en Internet</nombre>
    <codigo>PI-2026</codigo>
    <creditos>4</creditos>
  </informacion>
  <estudiantes>
    <estudiante id="001">
      <nombre>Juan</nombre>
      <apellido>Pérez</apellido>
      <email>juan.perez@example.com</email>
    </estudiante>
    <estudiante id="002">
      <nombre>María</nombre>
      <apellido>González</apellido>
      <email>maria.gonzalez@example.com</email>
    </estudiante>
  </estudiantes>
  <temas>
    <tema orden="1">HTML y CSS</tema>
    <tema orden="2">JavaScript</tema>
    <tema orden="3">XML y JSON</tema>
  </temas>
</curso>
```

<Note>
  This is just an example. You should create your own unique structure based on the information you want to represent.
</Note>

## What is Well-Formed XML?

A well-formed XML document must satisfy these requirements:

### XML Declaration
```xml
<?xml version="1.0" encoding="UTF-8"?>
```
While optional, it's considered good practice to include this at the beginning of your document.

### Single Root Element
```xml
<!-- Correct: One root element -->
<root>
  <child1>Content</child1>
  <child2>Content</child2>
</root>

<!-- Incorrect: Multiple root elements -->
<root1>Content</root1>
<root2>Content</root2>
```

### Proper Nesting
```xml
<!-- Correct: Properly nested -->
<parent>
  <child>Content</child>
</parent>

<!-- Incorrect: Overlapping tags -->
<parent>
  <child>Content</parent>
</child>
```

### Closed Tags
```xml
<!-- Correct: All tags closed -->
<element>Content</element>
<empty-element />

<!-- Incorrect: Unclosed tag -->
<element>Content
```

### Quoted Attributes
```xml
<!-- Correct: Quoted attributes -->
<element attribute="value">Content</element>

<!-- Incorrect: Unquoted attributes -->
<element attribute=value>Content</element>
```

<Warning>
  Pay special attention to special characters in XML. Characters like `<`, `>`, `&`, `"`, and `'` must be escaped using entity references when used in content or attributes.
</Warning>

## Validation Tools

You can use these tools to validate your XML:

- **Online Validators**: Search for "XML validator" to find free online tools
- **VS Code**: Install XML extensions that provide real-time validation
- **Command Line**: Use tools like `xmllint` (available on most systems)
- **Browser**: Modern browsers can parse and display XML files, showing errors if the document is not well-formed

<Info>

</Info>

## Common Mistakes to Avoid

- Forgetting to close tags
- Mixing up opening and closing tag names (case sensitivity)
- Not properly escaping special characters
- Having multiple root elements
- Incorrectly nesting elements
- Forgetting quotes around attribute values
