## Introduction

Practice assignments (Prácticas) are hands-on exercises designed to reinforce the concepts covered in the Internet Programming course. These assignments provide practical experience with web technologies, protocols, and programming techniques.

## How to Complete Assignments

<Steps>
  <Step title="Read the Assignment">
    Carefully review the assignment requirements and objectives for each practice.
  </Step>
  
  <Step title="Create a Branch">
    Create a new branch in the `prog-internet` repository following the naming convention:
    ```
    YourLastNameYourFirstName/PracticaNumber
    ```
    For example: `QuelaliGaston/Practica1`
  </Step>
  
  <Step title="Complete the Work">
    Implement the required functionality according to the assignment specifications.
  </Step>
  
  <Step title="Test Your Solution">
    Verify that your implementation meets all requirements and works correctly.
  </Step>
  
  <Step title="Submit Your Work">
    Commit and push your changes to your branch in the repository.
  </Step>
</Steps>

<Note>
  Always create a separate branch for each practice assignment. Do not work directly on the main branch.
</Note>

## Best Practices

### Repository Management

- **Use Descriptive Commits**: Write clear commit messages that explain what changes you made
- **Commit Frequently**: Make small, logical commits rather than one large commit
- **Follow Naming Conventions**: Stick to the prescribed branch naming format
- **Keep Branches Clean**: Only include files relevant to the specific practice

### Code Quality

- **Write Clean Code**: Use proper indentation and follow coding standards
- **Add Comments**: Document complex logic or important decisions
- **Test Thoroughly**: Verify your work before submitting
- **Validate Your Output**: Use appropriate tools to check your code (validators, linters, etc.)

<Warning>
  Make sure to follow the specific requirements for each assignment. Pay attention to file formats, naming conventions, and deliverable specifications.
</Warning>

## Submission Guidelines

1. Ensure all required files are included in your branch
2. Verify that your code is well-formed and follows specifications
3. Test your implementation thoroughly
4. Push your final changes to the repository
5. Confirm that your branch is visible in the remote repository

<Note>
  Submission deadlines will be announced in class. Plan your time accordingly and don't wait until the last minute.
</Note>
