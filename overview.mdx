## Learning Objectives

By the end of this course, you will be able to:

- Build interactive user interfaces with **React** and **TypeScript**
- Manage application state and handle asynchronous operations with **Axios**
- Create RESTful APIs using **Symfony** framework
- Design and implement database schemas with **Doctrine ORM**
- Containerize applications with **Docker** and **Docker Compose**
- Work with XML data structures and web service integration
- Apply modern JavaScript concepts including arrow functions, JSX, and array methods
- Debug and troubleshoot both frontend and backend applications

## How to Use This Repository

### Setting Up Frontend Projects

Frontend examples use **Vite** with React and TypeScript:

```bash
# Navigate to a React project
cd ejemplos/p1-react

# Install dependencies
npm install

# Start development server
npm run dev

# Build for production
npm run build
```

### Setting Up Backend Projects

Backend examples use **Docker Compose** for consistent environments:

```bash
# Navigate to the Symfony project
cd ejemplos/p4-symfony

# Build Docker images
docker compose build --pull --no-cache

# Start containers
docker compose up --wait

# Access at https://localhost

# Stop containers
docker compose down --remove-orphans
```

<Info>
  The Symfony Docker setup includes **FrankenPHP** with worker mode for blazing-fast performance, automatic HTTPS with Caddy, HTTP/3 support, and built-in Mercure hub for real-time features.
</Info>

### Working with Practices

Each practice includes instructions for creating branches and submitting work:

```bash
# Create a practice branch
git checkout -b YourName/Practica1

# Work on your solution
# ...

# Commit and push your work
git add .
git commit -m "Complete Practica 1: XML structures"
git push origin YourName/Practica1
```

## Key Concepts Reference

### JavaScript Fundamentals

The course builds on essential JavaScript concepts covered in the glossary:

- **Array Methods**: `.map()` for transforming data arrays into UI components
- **Arrow Functions**: Modern, concise function syntax with lexical `this` binding
- **Ternary Operators**: Conditional expressions for inline logic
- **Error Handling**: Using `try-catch-throw` for robust error management
- **JSX**: HTML-like syntax for describing React UIs (requires transpilation)

### React Patterns

- **Functional Components**: Modern React uses functions instead of classes
- **Hooks**: `useState`, `useEffect`, and custom hooks for state and side effects
- **Component Composition**: Building complex UIs from small, reusable pieces
- **Props & State**: Data flow and component communication
- **React Router**: Client-side routing for single-page applications

### Backend Architecture

- **MVC Pattern**: Symfony's Model-View-Controller structure
- **ORM**: Doctrine for database abstraction and entity mapping
- **Dependency Injection**: Symfony's service container for clean architecture
- **Routing**: Attribute-based routing for API endpoints
- **Form Handling**: Symfony forms with validation and CSRF protection
